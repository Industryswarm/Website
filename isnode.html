ISNode

Scalable Node.JS Framework for Enterprise Applications

Get Started Today

Loader

The Loader is responsible for loading all modules required for your services and managing their operation. It is the sole include and entry point for the application server.
Learn More

CLI Module

The CLI (or Command Line Interface) Module provides business logic to handle commands to the application server from the command line - as well as command-line help.
Learn More

Daemon Module

With ISNode there is no need to use Forever or PM2. The application server has daemon services baked right in to it so that you can run the server as a system daemon and monitor its state.
Learn More

Data

The Data Module provides your services with interconnectivity to a wide range of databases, including MySQL, PostgreSQL, MongoDB and more. It automatically loads and manages your models that you can then call upon from within each service.
Learn More

Error Handler Module

The Error Handler Module monitors all services running within the application server for unhandled/thrown errors - even if deep within the service's dependencies - and then gives those services the ability to handle those errors without terminating the application server.
Learn More

Identity Module

The Identity Module is a multi-tenant module for authentication and authorization. It supports organisations with many users, accessing multiple application profiles across many services. It has full support for OAuth2, SSO and IndieAuth, as well as a complete RBAC (Role Based Access Control) system.
Learn More

Jobs

The Jobs Module enables creation and management of asyncronous jobs. It supports job queues as well as recurring jobs (similar to Cron).
Learn More

Licensing Module

The Licensing Module provides you with the ability to create a license server and have your services ping this server at startup and during operation to validate licenses held within config. This is a foundational component for building commercial and re-distributable services, when combined with service compilation.
Learn More

Logger Module

The Logger Module provides an interface for all other modules within the application server - as well as services running on the server - to log debug, error and success messages. These messages can be rendered to the console, appended to a log file or pushed to a repository like ElasticSearch.
Learn More

Mailer Module

The Mailer Module provides you with the capability to build and push out emails from your services via SMTP.
Learn More

Router Module

One of the many interesting things about ISNode is that you can create many instances of any interface (whether it be HTTP, WebSockets, SSH or other) and define routes to one or more services running within the application server. The Router Module supports this functionality.
Learn More

Services Module

The ISNode Framework isn't just a framework, but also an application server. Within it you can run one service or many services. The Services Module is responsible for loading, enumerating and providing access to each of the potentially many services running within the server.
Learn More

Swagger Module

The Swagger Module provides an interface to access a Swagger file compliant JSON representation of all endpoints exposed within the services running on your application server.
Learn More

Utilities Module

The Utilities Module provides a range of useful utility methods that are used day to day in your services including - UUID and Random String Generation, Data Encryption/Decryption, Smart JSON Variable Parsing, Object Manipulation Methods, CSV Parsing and Building, and much more.
Learn More

HTTP Interface Module

The HTTP Interface Module allows you to create one or more HTTP or HTTPS interfaces within your application server, that can then be routed to the one or more services running within it.
Learn More

WebSockets Interface Module

The WebSockets Interface Module allows you to create one or more WebSockets interfaces within your application server, that can be bound to HTTP or HTTPS interfaces that are open, and then routed to the one or more services running within the server.
Learn More

SSH Interface Module

The SSH Interface Module allows you to create one or more SSH interfaces within your application server, that can then be routed to the one or more services running within it.
Learn More

ZeroMQ Interface Module

The ZeroMQ Interface Module allows you to create one or more ZeroMQ interfaces within your application server, that can then be routed to the one or more services running within it.
Learn More

NanoMSG Interface Module

The NanoMSG Interface Module allows you to create one or more NanoMSG interfaces within your application server, that can then be routed to the one or more services running within it.
Learn More
